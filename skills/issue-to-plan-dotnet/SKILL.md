---
name: issue-to-plan-dotnet
description: Convert a .NET bug report or feature request into an ordered implementation plan with validation steps.
---

## When to use
Use when the user asks for an implementation plan, task breakdown, or phased approach.

## Output
1. Goal
2. Assumptions
3. Constraints
4. Ordered tasks
5. Validation steps
6. Risks and follow-up work
