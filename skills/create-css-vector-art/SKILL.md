## When to use
Use when producing new interface art, iconography, decorative elements, or component-level illustrations that must stay vector-native and implementation-friendly.

## Output
- visual direction summary
- tokenized CSS/SVG shape and color plan
- responsive sizing strategy for key breakpoints
- accessibility and legibility checks for interface contexts

## Notes
- Keep final assets CSS/SVG/vector-native and avoid raster-dependent final outputs.
- Define reusable variables for color, stroke, spacing, radius, and elevation.
- Align the work with the active `AGENTS.md`, `ARCHITECTURE.md`, and any active roles.
- Prefer existing commands in `./commands/` when a deterministic workflow exists.
